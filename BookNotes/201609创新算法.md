很多问题一旦表述准确，就能很快找到解决办法。比如，从现象出发表述问题“滚珠轴承要定位在钻柄上，并且在切刀头放号之前不能掉下来”，换成抽象成系统要素的表述可以是“一个金属部件必须临时与另一个金属部件连在一起”。

我们生活在一个惯性思维的世界，在一般情况下惯性思维是有利的，但对于复杂情况，惯性思维就成为解决问题的障碍。人们不仅要学会如何有效的进行惯性思维，还要学会如何有效的打破惯性思维。

在很多发明的过程中，发明家首先识别出了有害因素（比如无法减轻的物体重量），发明往往不需要去掉有害因素，只是稍微修改它，让它变得有用。

### 创造性等级

创新过程会面对大量的混乱无序，何不开发一种按规则有序探索的过程呢 —— 启发法（Heuristics），开发通用原则来帮助解决问题。

创新过程的几个特征：

* 1级，使用一个已有的物体，不考虑其他物体
* 2级，在几个物体之间选择一个
* 3级，对选出来的物体做部分改变
* 4级，开发一个新物体，或者完全改变选择的物体
* 5级，开发一套全新的复杂系统

不要总是只去解决 1、2 级问题

* 1级发明，问题及答案存在于“某个专业领域”
* 2级发明，问题及答案存在于“某个行业领域”
* 3级发明，问题及答案存在于“某个学科领域”
* 4级发明，问题及答案存在于“问题起源的学科边界之外” 第二序
* 5级发明的较高子集，问题及其答案超出了现代科学的边界

1 级问题和 4 级问题之间的区别

1 级问题|	4 级问题
--- | --- 
问题的元素少|	问题有大量元素
没有或很少的未知元素|	有大量的未知元素
分析简单|	很难进行分析（很难分离已知和未知元素）
给定解决问题的时间短|	给定解决问题的时间很长

```
- 发明和创新过程级别不同，应该尽量去解决3-5级问题
- 用启发式方法为思考和创新提供合理而高效的路径
```

### 创新的创新方法
Osborn 发明的头脑风暴方法，一组过程只对问题提出想法，一组过程只分析想法。但是它没有消除搜寻的杂乱无章，反而使搜寻更加混乱。头脑风暴的成功，不是因为它有优点，只是传统试错法缺点太多。头脑风暴如果有任何收益，那就是能减少沿着惯性思维方向的无效尝试次数。头脑风暴的搜寻还是简单地把变量排列组合，无法超越 2 级发明。它不控制思维过程，这是它的主要缺陷。

有没可能为每个问题找到所有可能的变量列表？编排所有变量的完整列表方法叫“形态学分析”。

建立一个多维表（形态盒），它的参数轴是给定物体组合的主要特征。例如，可用的能量作为一个参数轴，运动引擎作为第二个参数轴，可用的推进元素类型作为第三个参数轴。组合数量太多是形态学的主要弱点，另外容易大片遗漏组合。优势是对参数轴和类别的直觉搜寻代替了对变量的直觉搜寻。（压缩、层级抽象、组块）

启发式问题法帮助我们一定程度上减少了心理惯性，但也就这样了。

综摄法：解决问题时，具有多样性背景的团队首先原封不动地接受问题，然后细化问题，把它变成所有人都能理解的结构。试图系统的从一个新角度看待问题，借此消除心理惯性。四种类比方法：

- 直接类比：与自然界或另一个领域类似主题相比较
- 拟人类比：把自己想象成系统
- 象征类比：通用化和抽象化
- 虚拟类比：把一些假想的小生命引入问题中

综摄法的局限：没有研究技术系统计划的客观规律。

### 通过知识而不是数量

如果世界上真的有直觉这回事，那么就一定存在某种形式的潜规则。科学的目标就是揭示这些潜规则。

一个创新问题可以不通过简单试错，而是正确地组织创新过程，用结构化的思考过程来解决。

### 理想机器

需要创新的问题从不匮乏，我们需要判断什么问题必须解决。发明家绝对不能接受别人编排好的问题陈述。任何正确陈述的问题，可能早就被第一个遇到它的人解决了。

技术系统进化的主要方向之一，就是改变系统的规模。一般而言，运输和加工的机器变得越来越大，控制和测量设备变得越来越小。

理想机器的特征：

- 重量、体积、面积正好和与之产生交互作用的物体的重量、体积、面积一样大
- 机器的所有部件以最大的能力做有用功

结构化的搜寻能够将一个人的思维过程有机组织起来，从而提高效率。

### 技术矛盾
设计师选择最有利的一组特性参数进行设计。很大程度上，设计师的艺术取决于平衡得失的技能。当问题有额外的要求——有得没有失，创新的需求就产生了。

从工程的角度看，一项创新通常表明全部或部分地克服了一个技术矛盾。

> 创新就是消除技术矛盾


## 第二章 创新的辩证法

### 循序渐进
理想机器能确定搜寻的方向，而技术矛盾则指出必须克服的障碍。

数学上，算法是解决问题的一系列严谨的步骤。广义上讲，算法由**一系列有序的结构化活动**组成。

#### ARIZ （ Algorithm of TRIZ ）61

**第一阶段——分析**

- 步骤1：陈述问题
- 步骤2：设想最终理想解
- 步骤3：确定是什么妨碍了取得这个结果（找到矛盾）
- 步骤4：确定为什么会妨碍得到这个结果（找到矛盾的原因）
- 步骤5：确定在什么条件下不会妨碍得到这个结果（找到消除矛盾的条件）

**第二阶段——实施**

- 步骤1：改变物体（给定的机器，设备和/或技术流程）本身。尺寸，形状，材料，温度，压力，速度，颜色，部件的相对位置，部件的工作条件使工作负载最大化
- 步骤2：把物体分成几个独立的部分。隔离“弱的”部件；隔离“必要的/充分的”部件；把一个物体分成几个相同的部件
- 步骤3：改变（给定物体的）外部环境。改变环境参数；替换环境；把环境分成几种介质；利用环境的特征来实现有用功能
- 步骤4：改变（相互作用的）相邻物体。定义参与同样功能的独立物体之间的关系；把功能转移到其他物体，消除这个物体；利用反面的闲置空间，增加在某一区域上同时工作的物体数量
- 步骤5：研究其他领域的原型（在另一个技术领域怎么解决类似矛盾？）
- 步骤6：回到最初的问题，放宽条件——转换到更一般性的问题陈述

**第三阶段——综合**

- 步骤1：改变给定物体的形状——新功能的机器应该有新形状
- 步骤2：改变与这个物体相互作用的物体
- 步骤3：改变这个物体起作用的方式
- 步骤4：应用解决其他技术问题的新原理

#### ARIZ （ Algorithm of TRIZ ）71

**第一阶段——选择问题**

- 步骤1-1：确定答案的最终目标
	+ 技术目标是什么（物体必需改变的特征是什么）？
	+ 在解决问题的过程中，明显不能改变的特征是什么？
	+ 答案的经济目标是什么（如果问题解决了，能减少哪方面成本）？
	+ 大概可以接受的成本是什么？
	+ 必需改善的主要技术或经济特征是什么？
	
- 步骤1-2：尝试“变通方法”：假设这个问题从根本上不能解决，那么解决哪些一般性问题可以达到最终结果？
- 步骤1-3：初始问题或变通问题，那一个解决起来更有意义：
	+ 将初始问题与给定行业内的一个趋势（进展方向）相比
	+ 将初始问题与领先行业内的一个趋势（进展方向）相比
	+ 将变通问题与给定行业内的一个趋势（进展方向）相比
	+ 将变通问题与行业内的一个领先趋势（进展方向）相比
	+ 将初始问题和变通问题对比，选择其中一个进行研究
- 步骤1-4：确定量化特征
- 步骤1-5：对这个量化特征引入时间校正
- 步骤1-6：定义让发明起作用的特殊条件要求

**第二阶段——精确地定义问题**

- 步骤2-1：用专利信息更精确的定义问题
	+ 在其他专利中解决的问题，与给定的问题有多接近？
	+ 在领先行业中已经解决的问题，与给定的问题有多相似？
	+ 相反的问题是怎么解决的？
- 步骤2-2：使用 STC 算子（Scale，Time，Cost）
	+ 假定改变物体的尺寸，从给定值到 0，这个问题能解决吗？如果可以，怎么解决？
	+ 假定改变物体的尺寸，从给定值到无穷大，这个问题能解决吗？how？
	+ 假定改变过程的时间（或者物体的速度），从给定值到 0，这个问题能解决吗？how？
	+ 假定改变过程的时间（或者物体的速度），从给定值到无穷大，这个问题能解决吗？how？
	+ 假定改变物体或过程的成本，从给定值到 0，这个问题能解决吗？how？
	+ 假定改变物体或过程的成本，从给定值到无穷大，这个问题能解决吗？how？
- 步骤2-3：用两句话描述问题的条件
	+ “给定一个系统，由什么部件组成”，例如：“一个管道，有一个阀门”
	+ “部件在什么条件下产生不希望的结果”，例如：“带铁矿颗粒的水通过管道运输，铁矿颗粒会磨损阀门”
- 步骤2-4：把2-3的部件列入下表
	
	部件类型|部件
	---|---
	a.能够改变、重新设计或者重新调整的部件；b.很难改变的部件|a：管道，阀门；b：水，铁矿颗粒
- 步骤2-5：选择最容易的部件，改变、重新设计或者调整
	+ 如果所有部件改变的难易程度一样，就从一个不动件开始
	+ 如果一个部件与不良效果联系在一起，最后才考虑这个部件
	+ 如果系统部件都很难改变，那么从外部环境中选择一个部件

**第三阶段——分析**

- 步骤3-1：归纳最终理想解
	+ 从步骤2-5中选择一个部件
	+ 陈述它的活动；
	+ 陈述它如何完成这个活动
	+ 陈述它何时完成这个活动
	+ 陈述在什么条件下它完成这个活动

- 步骤3-2：画最终理想解之前的“初始图”和之后的“理想图”。步骤2-3a 中所有部件都必需出现在图中
- 步骤3-3：在“理想图”中，找到步骤3-1a 指出的部件，并且把那些在规定的条件下不能实施规定功能的部分，重点标出来
- 步骤3-4：为什么这个部件自己不能完成规定活动？
	+ 从物体重点标记的地方我们期望得到什么？
	+ 什么妨碍它自己完成这个活动？
	+ 上述两个问题之间有什么冲突？
- 步骤3-5：在什么条件下，这个部件能够完成规定的活动（应该有什么参数？）
- 步骤3-6：为了让这个部件得到3-5中描述的特征，需要做什么？
	+ 在图上，在物体的标记区用箭头画出所需施加的外力，以实现需要的特征
	+ 怎样产生这些外力？
- 步骤3-7：归纳所有能够实现的概念
- 步骤3-8：画出原理图，实现概念1（完成后进行下一个概念）
	+ 新设备中工作部件的“聚合”状态是什么？
	+ 在一个循环内，设备如何变化？
	+ 多次循环后，设备如何变化？

**第四阶段——概念的初步分析**

- 步骤4-1：在应用新概念的时候，什么变好了，什么恶化了，记录得到了什么，什么变得更复杂或者更昂贵了？
- 步骤4-2：改变提出的设备或者方法，能否防止其恶化？用图标是这个设备或者方法
- 步骤4-3：现在改变了的设备什么恶化了（更复杂，更昂贵）？
- 步骤4-4：比较得失。哪一个更大？为什么？如果得大于失，跳到第六阶段。如果失大于得，返回3-1.
- 步骤4-5：如果第二次分析没有产生新结果，返回2-4检查表格。从2-5选择系统的其他部件，重新分析。如果仍没有满意答案，进入阶段五

**第五阶段——实施**

- 步骤5-1：从矛盾矩阵的列中，选择一定要改善的特征
- 步骤5-2：a.使用已知的手段（不考虑其他方面损失）改善这个特征；b.如果采用已知手段，什么特征变得不可接受了？
- 步骤5-3：从矛盾矩阵的行中，选择与5-2b 相应的特征
- 步骤5-4：在矩阵中，找到用来消除技术矛盾的原理
- 步骤5-5：如何使用这些原理。如果问题解决，回到第四阶段，然后跳到第六阶段。如果没有，那么继续下面步骤
- 步骤5-6：尝试应用物理现象和效应
- 步骤5-7：尝试改变活动的时刻或持续时间
	+ 能否“延长”活动的时间来消除矛盾？
	+ 能否“缩短”活动的时间来消除矛盾？
	+ 能否在物体开始操作之前，提供一个活动来消除矛盾？
	+ 能否在物体开始操作之后，提供一个活动来消除矛盾？
	+ 如果过程是连续的，能否把它转变成周期性的？
	+ 如果过程是周期性的，能否把它转变成连续的？
- 步骤5-8：在自然界中，类似的问题是如何解决的？
	+ 自然界的非生命体如何解决这个问题？
	+ 古代的动植物如何解决这个问题？
	+ 现代的有机物如何解决这个问题？
	+ 在考虑特定的新技术和材料时，必须做哪些修正？
- 步骤5-9：尝试改变那些与我们研究的物体协同工作的物体
	+ 我们的系统属于哪个超系统？
	+ 如果我们改变超系统，这个问题如何解决？
	如果问题仍没有解决，返回1-3，如果解决返回第四阶段，评估想法，然后继续第六阶段
	
**第六阶段——综合**

- 步骤6-1：确定如何改变我们修改的系统所属的超系统
- 步骤6-2：探索如何用不同的方式应用已经修改的系统
- 步骤6-3：应用新发现的技术想法（或与之相反的想法），来解决其他技术问题。

### 逻辑、直觉和技巧的有机结合

在运用创新算法时，发明家首先要全面的分析问题，然后一步一步剥去不具体的外层，聚焦问题的本质。解决问题成功与否，主要取决于把系统从原始心理意象中“释放”出来的技能高低。创新算法的第二阶段，就是这个“释放”程序。

心理惯性不仅产生于描述物体的术语，也产生于物体时间、空间和成本的习惯性想象。STC 算子是一个序列化的心理试验，它帮助人们克服对物体的传统意象。它的目的是产生几个指向答案方向的想法，帮助克服分析问题时的心理障碍。

```
收获：
- 创新有几个前提：正确定义问题，发现技术矛盾，抽取模块并变换组合以产生新的解法
- 创新想法需要克服思维定式（心理惯性），时间、空间、成本是常见的心理意向维度
- 创新算法目的是提供全面的框架，将创新过程变成一系列可执行的操作，就像计算机程序一样
```

### 古生代的“专利品”
大自然独特而巨大的优越性，是设计控制和测量设备的灵感源泉。

在选择最完美的自然原型时，我们都使用最新的自然专利库。这就难怪自然界的技术进化看来不可理解，因为我们是倒着阅读它的！更值得使用的原型，是哪些自然界中相对不完美，但是更简单的“专利”，比如古生物学研究的那些古代动物。

ARIZ-71 建议发明家不仅要找到一个古老的原型，还应确定自然设计的进化路径。

## 第三章 人和算法
### 心理障碍
实际上，当问题出现时，解决它的条件已经具备了。

“不可能”的出现，就是因为人们不知道它是怎么发生的，因此，他们事先就说这一般是做不到的。但是，我们必须假定这是可以做到的，只是我们还不完全了解如何去做。

尝试用通用词汇替代技术词汇。例如，管子替代管道。

### 幻想的力量
没有系统地开发出一套幻想的方法。

科幻作品帮助我们克服通向“疯狂”想法道路上的心理障碍。幻想的影响就在于它对真正的工作思路起反作用。

###  创造性工作的科学结构
研究“领先”行业，研究“跟随者”行业的技术领域（主要用于创造性过程的综合阶段），收集关于物理效应、新材料、解决技术问题的方法等信息，研究专利信息，跟踪发明创造理论的文献，在解决练习问题中积累经验。

创新算法的本质是向发明家提供在更高级别上有用的经验。创新算法一定要让思考的过程更富有技巧，一定要提供符合需要并能可靠起作用的可控“直觉”。
